# 自定义404页面

本文档介绍如何在GHProxy中配置和使用自定义404页面功能。

## 功能概述

自定义404页面功能允许您：

1. 使用内置的404错误页面（默认行为）
2. 使用您自己设计的自定义404页面

## 配置方法

在`config.toml`配置文件中，找到`[pages]`部分，添加或修改`custom404`选项：

```toml
[pages]
mode = "internal" # "internal" or "external"
theme = "bootstrap" # "bootstrap" or "nebula"
staticDir = "/data/www"
custom404 = "" # 自定义404页面路径，为空则使用内置404页面
```

### 配置选项说明

- `custom404`: 指定自定义404页面的文件路径
  - 留空: 使用内置的404页面（默认行为）
  - 设置为有效的HTML文件路径: 使用指定的自定义404页面

## 使用示例

### 使用内置404页面

```toml
[pages]
custom404 = "" # 留空使用内置404页面
```

### 使用自定义404页面

```toml
[pages]
custom404 = "/path/to/your/custom-404.html" # 指定自定义404页面路径
```

## 自定义404页面示例

项目中提供了一个示例404页面文件 `example-404.html`，您可以基于此文件进行修改，创建自己的自定义404页面。

## 注意事项

1. 自定义404页面必须是有效的HTML文件
2. 如果指定的自定义404页面文件不存在或无法读取，系统将自动回退使用内置404页面
3. 自定义404页面仅适用于404错误，其他错误码仍使用内置错误页面

## 工作原理

当用户请求的资源不存在时，系统会：

1. 检查是否配置了自定义404页面
2. 如果配置了且文件存在，则返回自定义404页面
3. 如果未配置或文件不存在，则返回内置404页面

这种设计既保证了系统的稳定性，又提供了足够的灵活性，允许用户根据自己的需求自定义错误页面。
